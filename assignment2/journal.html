Comments:

Single line comments start with // and finish at the end of the line of code:
// this is a short comment

Multiline comments start with /* and end with */:
/* This is a longer comment
but anything on the lines between
the comments will be ignored.
*/

Grammar:

ASI: Automatic Semicolon Insertion
This means that each line of code with assume that it has a ; at the end.
However, some validators and such won't work without all the appropriate semicolons.
Best Practice is to end each statement with a semi-colon and start a new line.

Reserved Words:

This list of words can not be used for variable names, function parameters, object properties etc:
These words are reserved for use JS language:
abstract, await, boolean, break, byte, case, catch, char, class, const, continue, debugger, default, delete, do, double, else, enum, export, extends, false, final, finally, float, for, function, goto, if, implements, import, in instanceof, int, interface, let, long, native, new, null, package, private, protected, public, return, short, static, super, switch, synchronized, this, throw, throws, transient, true, try, typeof, var, volatile, void, while, with, and yield
Best Practice is to also not use undefined, NaN, and Infinity.

Primitive Data Types:

JavaScipt has 7 data types:
Six are Primaitve Data Types: String, Symbol, Number, Boolean, Undefined, and Null
If any value isn't one of these it will be an object
To figure out the type of a value you can use typeof and it will return the type of data type.

Declaring and Assigning:

From ES6 onward, JavaScipt uses const and let to declare variables. These are blocked scoped, meaning their value only exists insid eth block they are declared in.
const is used to assign a variable that will not be reassigned any other value.
let is used if the variable might be reassigned later in the program.
var is no longer Best Practice. You can also assign multiple values at once using commas. Ex: let x=3, y=4, z=5;
If a variable references an object using const doesn't make it unchangable. The data in an object can change/mutate.

Ex:
const name = { value: 'Alexa' }; // an object
name.value = 'Siri'; // change the value
<< 'Siri'

However, using const on an object means you can't reassign a variable to another object.

Ex: // declare object
const name = { value: 'Alexa' };
// attempt to reassign to another object
name = { value: 'Siri' }
<< TypeError: Assignment to constant variable.

Other Must knows:

Constant and varaible names can start with an upper or lower-case letter, an underscore, or dollar character. You can also use letters but can't start with a number.
Avoid starting with an underscore and using the $ character for other programs and frameworks use them.
Be consistent! Using Camel case is Best Practice. Start with lowercase letter and then uppercase at the start of every new word. Also use underscoreds to separate each new word.



Backslashes: must be used to escape special character in strings.

  • sing quote marks: \'
  • double quote marks: \"
  • end of line: \n
  • carriage return: \r
  • tab: \t
  • backslash: \\



Dot Notation:

Access properties of a string by writing a dot followed by the property we are interested in. Such as the length property.
variable.length returns the length of the string.



Don't forget about methods! You can use dot notation to use them and they end in (). This allows you to manipulate a string and gain info about them. If the thing you are looking for returns a -1 then they don't appear in the string. Method includes() returns ture or false if character you're looking for is included in the sting or not.



The concat() method can concatenate two or more strings together.Ex: 'JavaScript'.concat('Ninja'); returns 'JavaScriptNinja'
Or the short hand is to use + between strings. Ex: 'Java' + 'Script' + '' + 'Ninja'; returns 'JavaScript Ninja'



Template Literals:

template literals are a special string using the backtick character `` which makes it possible to use both sets of quote in the string '' and "",br> They also allow interpolation of JavaScipt code. Meaning a JavaScipt expression can be inserted inside a string and the result will be displayed:
Ex:
const name = `Siri`;
`Hello ${ name }!`;
<< 'Hello Siri!'

const age = 39;
`I will be ${ age + 1 } next year`;
<< 'I will be 40 next year'

The JavaScript is place inside the curly braces with a $ character infront of them. You can include line breaks with \n in a string.
